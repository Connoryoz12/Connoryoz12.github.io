Active Directory & Endpoint Security Lab

← Back to Main Page

Overview

This lab focuses on building and managing a Windows Active Directory environment and practicing endpoint security techniques. I deployed a small enterprise-style AD domain, created users and groups, configured Group Policy Objects (GPOs), and implemented endpoint monitoring and hardening. The goal is to gain hands-on experience with centralized identity management, security auditing, and incident response workflows.

Setup

Virtualization Platform: Oracle VirtualBox running Windows Server (Domain Controller) and multiple Windows 10 client VMs.
Directory Services: Windows Server Active Directory with DNS and DHCP configured for the internal lab network.
Endpoint Security: Sysmon for advanced logging, local firewall rules, and audit policies on endpoints.
Network Configuration: Internal VirtualBox network simulating a small enterprise environment with host-only adapter.

Tools Used

  • Windows Server Active Directory & Group Policy Management.
  • Sysmon and Windows Event Viewer for detailed security logging.
  • PowerShell for automation and reporting.
  • Microsoft Security Baselines for hardening guidance.
  • MITRE ATT&CK framework to map and test detection scenarios.

Objectives

  • Deploy a functioning Windows Active Directory domain with multiple endpoints.
  • Create and manage user accounts, organizational units, and group policies.
  • Implement endpoint monitoring to detect suspicious activity.
  • Practice security hardening and basic incident response procedures.

Steps

  1. Deployed a Windows Server VM and promoted it to a Domain Controller with DNS and DHCP roles.
  2. Joined multiple Windows 10 client VMs to the domain.
  3. Created organizational units, user accounts, and groups to simulate an enterprise structure.
  4. Configured Group Policy Objects (GPOs) to enforce security baselines (password policies, firewall rules, logging).
  5. Installed Sysmon on endpoints for enhanced event logging.
  6. Monitored domain and endpoint logs for failed logins, privilege escalations, and unusual process creation.
  7. Practiced incident response workflows using simulated attacks (unauthorized login attempts, lateral movement).

Findings / Results

Successfully implemented a Windows AD domain with multiple clients and enforced security through GPOs. Detected and analyzed suspicious user activity and abnormal process execution via Sysmon and Event Viewer. Documented findings and recommended additional hardening steps (multi-factor authentication, stricter audit policies, and endpoint isolation procedures). Demonstrated real-world skills in Active Directory administration, endpoint security, and basic SOC-style monitoring.
