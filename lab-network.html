Network Traffic & Threat Detection Lab

← Back to Main Page

Overview

This lab focuses on capturing and analyzing network traffic in a controlled environment to detect threats, intrusions, and malicious activity. Using Wireshark and Snort, I built a virtual lab to observe normal and abnormal network behaviors. The purpose of this lab is to gain hands-on experience with packet analysis, intrusion detection, and threat hunting techniques.

Setup

Virtualization Platform: Oracle VirtualBox running multiple virtual machines (Windows Server, Ubuntu Server, and Windows 10 client).
Network Monitoring Tools: Wireshark for packet capture and Snort for intrusion detection.
Network Configuration: Internal VirtualBox network with host-only adapter; controlled test traffic generated between VMs to simulate attacks and normal traffic.
Endpoints: Windows and Linux VMs with configured services to generate network events.

Tools Used

  • Wireshark for packet capture and deep packet inspection.
  • Snort as an IDS to analyze traffic against predefined rules.
  • Nmap and custom scripts to generate port scans and test traffic.
  • Windows and Linux utilities (netcat, curl, etc.) to simulate C2 and malware activity.
  • MITRE ATT&CK for threat detection mapping.

Objectives

  • Capture and analyze real-time network traffic in a lab environment.
  • Detect simulated port scans, C2 traffic, and malware communication.
  • Develop Snort rules to identify suspicious behavior.
  • Document findings and recommend mitigation strategies.

Steps

  1. Deployed Windows and Linux VMs on Oracle VirtualBox.
  2. Configured an internal network to safely route traffic between VMs.
  3. Installed Wireshark on a monitoring VM to capture packets.
  4. Installed and configured Snort with custom rules to detect attacks.
  5. Used Nmap to simulate port scans and network reconnaissance.
  6. Generated mock malware/C2 traffic using netcat and curl to test detection.
  7. Analyzed captured packets and IDS alerts to identify anomalies.

Findings / Results

Successfully captured and analyzed network traffic to identify simulated port scans and C2 communication. Snort rules triggered alerts for suspicious behavior, demonstrating the effectiveness of IDS in a lab setting. Documented findings, created a threat report, and recommended hardening measures (firewall rules, network segmentation, and IDS tuning).
