SIEM & Log Analysis Lab

← Back to Main Page

Overview

This lab simulates a real-world Security Operations Center (SOC) environment to develop hands-on skills in log collection, analysis, and alerting. Using the ELK Stack and Wazuh, I built a self-contained SIEM platform to monitor Windows and Linux endpoints. The purpose of this lab is to practice detecting unauthorized access attempts, brute-force activity, and suspicious processes in a safe environment while improving my knowledge of dashboards, queries, and security event correlation.

Setup

Virtualization Platform: Oracle VirtualBox running multiple virtual machines (Windows Server, Ubuntu Server, and Windows 10 client).
SIEM Platform: ELK Stack (Elasticsearch, Logstash, Kibana) with Wazuh for endpoint integration.
Endpoints: Windows 10 workstation and Ubuntu Linux server configured to forward security logs.
Network Configuration: Internal network in VirtualBox to simulate a SOC environment; host-only adapter to allow the host machine to interact with lab machines safely.

Tools Used

  • Elasticsearch & Kibana for storing and visualizing log data.
  • Logstash for parsing and enriching logs.
  • Wazuh Agent installed on endpoints for real-time log forwarding.
  • Sysmon on Windows endpoints for detailed event collection.
  • PowerShell & Bash scripts to generate test events.
  • MITRE ATT&CK as reference for detection use cases.

Objectives

  • Build a functioning SIEM environment from scratch.
  • Forward and normalize logs from Windows and Linux endpoints.
  • Create custom dashboards in Kibana to visualize failed logins, process creation, and network anomalies.
  • Detect simulated malicious activities and practice incident response reporting.

Steps

  1. Installed and configured ELK Stack and Wazuh on an Ubuntu server VM in VirtualBox.
  2. Installed Wazuh agents on Windows and Linux endpoint VMs.
  3. Configured Sysmon on Windows endpoints to collect detailed event logs.
  4. Forwarded security logs to the SIEM for real-time analysis.
  5. Created Kibana dashboards for authentication attempts, new process creation, and network connections.
  6. Generated test events using brute-force scripts and simulated malware execution.

Findings / Results

Successfully detected brute-force login attempts on the Windows VM and alerted via Wazuh rules. Created correlation searches to identify abnormal process execution. Documented alerts, created an incident report, and recommended mitigation actions (account lockouts, GPO hardening). Demonstrated ability to stand up a complete SIEM environment, collect logs from multiple OSs, and visualize threats effectively.
